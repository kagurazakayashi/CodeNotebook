# 常见语言的代码检查静态分析命令

| 语言 / 生态 | 核心检查命令                 | 主要用途                                | 工具类型           |
| :---------- | :--------------------------- | :-------------------------------------- | :----------------- |
| **Flutter** | `flutter analyze`            | 检查 Dart 代码、类型安全及 Flutter 规范 | 官方集成           |
| **Python**  | `ruff check`                 | 2026 年最推荐，极速替代 Flake8/Pylint   | Linter (Rust 编写) |
| **Python**  | `mypy .`                     | 专门用于检查 PEP 484 静态类型           | Type Checker       |
| **JS / TS** | `npx eslint .`               | 前端行业标准，检查逻辑错误与风格        | Linter             |
| **JS / TS** | `npx @biomejs/biome check .` | 极速全能工具 (检查 + 格式化)            | Linter / Formatter |
| **Rust**    | `cargo check`                | 快速编译检查（不生成可执行文件）        | Compiler Tool      |
| **Rust**    | `cargo clippy`               | 官方推荐，提供代码优化与惯用法建议      | Static Analysis    |
| **Go**      | `go vet ./...`               | 检查代码中可疑的结构（如错误的 Printf） | Built-in Linter    |
| **Go**      | `golangci-lint run`          | 社区最强工具，集成数十种检查器          | Linter Aggregator  |
| **Java**    | `mvn checkstyle:check`       | 验证代码是否符合定义的编码规范          | Style Checker      |
| **Kotlin**  | `ktlint`                     | 反馈 Kotlin 官方风格指南的检查工具      | Linter             |
| **Swift**   | `swiftlint`                  | iOS/macOS 开发中强制执行风格规范        | Linter             |
| **C / C++** | `cppcheck .`                 | 检查内存泄露、越界等深度逻辑错误        | Static Analysis    |
| **C / C++** | `clang-tidy`                 | 基于 LLVM 的强大静态分析工具            | Static Analysis    |
| **PHP**     | `phpstan analyse`            | 强类型分析，专门寻找潜在的代码 Bug      | Static Analysis    |
| **Ruby**    | `rubocop`                    | 检查 Ruby 社区最佳实践指南              | Linter             |
